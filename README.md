这四个实验太见鬼了，不清楚的告诉你自己的测试到底涵盖了哪些范围，只能一点点猜，不参考前人的代码几乎无法正常通过

具体的使用方法如下：

### lab1

- **提交的文件**

  - `lex_main.c`
  - `lex.l`
  - 你的文档

- **本地测试**：

  - **编译：**`make`

    生成的 `lexer` 就是 可执行文件

  - **运行测试：**

    - **测试样例是题目的：**

      - ```c
        int main(){
            return 3;
        }
        ```

    - **命令**

      - `echo 'int main(){ return 3;}' | ./lexer`

# 关于测试

如果你希望在本地研究一下这些代码，那么我假定你对一些基础的工具已经有了认识，在每个目录下都有一个makefile，通过执行

```sh
make
```

就能够顺利的得到可运行的二进制文件，可实现每个实验要求的功能,如果需要清除生成的乱七八糟的文件，可以在目录下执行

```shell
make clean
```



此外，还需要提醒的是，要实现上述流程，需要确保你的设备安装了flex,bison,gcc,make.至于这些工具到底在那里找，请询问ai
